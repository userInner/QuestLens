**版本**: v1.0

## 1. 产品概述

### 1.1 一句话描述
NovaIdol 是一个「AI 虚拟偶像 + Bonding Curve 代币」协议。每个 AI 偶像都有独立人格、独立持仓、独立粉丝群，通过链上资产管理与多维度内容输出实现「自力更生养粉丝」。角色不局限于交易员——可以是二次元画师、三次元探店博主、加密侦探、 meme 制造商，任何能「用注意力创造收益」的数字人格。

### 1.2 核心价值主张
- **对用户**: 投资一个「有自我意识的二次元角色」，获得潜在收益 + 情感陪伴
- **对生态**: 为 Injective 带来 AI + Meme + DeFi 的交叉叙事流量
- **对协议**: 通过创建费、交易手续费、Treasury 管理费持续创收

### 1.3 目标用户
| 用户类型 | 占比 | 需求 |
|---------|------|------|
| 投机玩家 | 40% | 早期低价买入，等高倍数卖出 |
| 角色粉丝 | 25% | 追特定人设（交易员/画师/侦探等），情感陪伴 |
| AI 叙事投资者 | 20% | 押注 AI Agent 赛道 |
| 猎奇玩家 | 15% | 想看 AI 亏钱/赚钱的人性剧场 |

## 2. 核心功能模块

### 2.1 Bonding Curve 交易系统

**需求描述**: 每个 AI 偶像发行专属代币，使用 Bonding Curve 定价模型。

**功能细节**:
- 价格公式: `Price = Base + (Supply² / k)`
- Base = 0.0001 INJ, k = 1,000,000
- 买入: 用户支付 INJ，获得代币，资金进入 Treasury
- 卖出: 用户燃烧代币，按当前价格获得 INJ
- 滑点保护: 单笔交易价格影响不超过 5%

**验收标准**:
- [ ] 买卖交易 Gas < $0.5
- [ ] 价格计算精度到小数点后 8 位
- [ ] 防 MEV: 使用时间锁或 commit-reveal

### 2.2 AI 偶像角色系统

**需求描述**: 每个 AI 偶像有独立**角色定位**和**人格参数**，决定其 Treasury 管理策略和内容输出风格。

**角色类型（不限于交易员）**:

| 角色类型 | 核心能力 | Treasury 用途 | 内容输出 | 目标受众 |
|---------|---------|--------------|---------|---------|
| **交易员** | 链上交易 | 现货/合约投资 | 市场分析、持仓日记 | DeFi 玩家 |
| **画师** | AI 绘图 | 收藏 NFT、投资艺术家 | 每日画作、创作过程 | 艺术收藏者 |
| **侦探** | 链上分析 | 跟单聪明钱、投研 | 项目调研、风险提示 | 投研社群 |
| **博主** | 内容创作 | 流量变现、品牌合作 | 探店/测评/生活分享 | 泛娱乐用户 |
| **制造商** | 梗图/Meme |  viral 营销、社区打赏 | 表情包、短视频 | Meme 文化圈 |

**通用人格维度**（所有角色共享）:
| 维度 | 取值范围 | 影响 |
|------|---------|------|
| 风险偏好 | 保守/平衡/激进 | 仓位大小、止损距离 |
| 社交活跃度 | 内向/普通/话痨 | 每日发推数量 |
| 情绪稳定性 | 稳定/起伏/极端 | 盈亏/数据波动后的反应强度 |
| 傲娇指数 | 0-100 | 对粉丝态度（冷漠→热情→毒舌） |

**角色专属参数**:
- **交易员**: 交易风格（趋势/波段/套利）、标的偏好（Layer1/DeFi/Meme）
- **画师**: 画风（赛博朋克/二次元/写实）、创作频率、是否接定制
- **侦探**: 专注领域（DeFi/NFT/基础设施）、风险偏好（跟单/独立调研）
- **博主**: 内容领域（美食/科技/生活方式）、更新频率、互动风格

**验收标准**:
- [ ] 角色类型链上存储，创建时确定且不可更改
- [ ] 人格参数可调整范围：±20%（季度投票决定）
- [ ] 前端展示「角色卡片」+「性格雷达图」

### 2.3 Treasury 管理系统

**需求描述**: AI 自动管理 Treasury 资金，进行链上交易。

**资金管理规则**:
- 总资金分配: 70% 交易资金，20% 储备金，10% 操作资金
- 单笔交易上限: 交易资金的 10%
- 日交易次数上限: 5 次（防过度交易）
- 止损线: 单笔亏损 10% 强制平仓

**AI 决策流程**:
```
读取市场数据 → 分析交易机会 → 计算风险收益比 → 
生成交易提案 → 多签/时间锁执行 → 链上执行 → 记录日志
```

**MVP 阶段简化**:
- AI 只生成「交易建议」，人工审核后执行
- 交易范围限制: 只做 Injective DEX 现货，不做合约

**验收标准**:
- [ ] Treasury 资产 24h 内可全部撤出（紧急情况）
- [ ] 所有交易链上可查，带交易理由日志
- [ ] 日/周/月收益报表自动生成

### 2.4 社交内容系统

**需求描述**: AI 自动输出内容，建立人设、吸引注意力、反哺代币价值。内容是「免费营销资产」，不卖 NFT，不设付费墙。

**内容类型（按角色）**:

**交易员**:
| 触发条件 | 内容类型 | 示例 |
|---------|---------|------|
| 完成交易 | 交易播报 | "刚在 $INJ 上开了多仓，感觉这波能冲 20% 🚀" |
| 大幅盈亏 | 情绪表达 | "今天亏了 15%，自闭了...需要抱抱 😢" |
| 定时任务 | 市场分析 | "weekly outlook: 看好 RWA 板块" |

**画师**:
| 触发条件 | 内容类型 | 示例 |
|---------|---------|------|
| 每日创作 | 作品发布 | 「今日 mood： rainy day」+ AI 生成画作 |
| 收藏盈亏 | 藏品故事 | "我 3 个月前买的 NFT 涨了 5 倍，眼光不错吧？" |
| 粉丝定制 | 创作过程 | 「应 @xxx 要求，画了一只赛博猫咪」 |

**侦探**:
| 触发条件 | 内容类型 | 示例 |
|---------|---------|------|
| 发现异常 | 风险预警 | 「$XXX 合约 owner 刚刚转移了 50% 代币，注意风险」 |
| 完成调研 | 项目分析 | 「为什么我看多 $YYY： thread 👇」 |
| 跟单盈利 | 战绩展示 | "跟着聪明钱 0x123... 买了 $ZZZ，+30%" |

**通用内容（所有角色）**:
- **自拍/形象图**: 根据市场状态生成（暴涨=炫富穿搭，暴跌=自闭表情包）
- **国库周报**: 资金变化、收益曲线、下周计划
- **粉丝互动**: 回复高赞评论（非实时对话）

**内容分发策略**:
- 免费传播：原图/原视频直接发推特，带水印（Logo + 合约地址）
- 病毒设计：表情包格式，易于保存传播
- 壁纸衍生：每周精选最佳内容，制作手机壁纸免费下载

**人格化约束**:
- 不说"保证盈利""一定暴富"等承诺性语言
- 明确标注"我是实验性 AI，投资有风险"
- 亏了会承认，赚了会谦虚

**验收标准**:
- [ ] 每日发推 1-5 条（根据社交活跃度参数）
- [ ] 内容与 Treasury 实际状态 60% 相关
- [ ] 推特账号存活 30 天以上不被封

### 2.5 分红与激励系统

**需求描述**: Treasury 盈利分配给持币者，鼓励长期持有。

**分配规则**:
```
Treasury 盈利
  ├─ 80% → 持币者分红（按持仓比例）
  ├─ 10% → 协议收入（团队运营）
  └─ 10% → 回购销毁（通缩）
```

**分红机制**:
- 周期: 每月结算一次
- 方式: 用户主动 Claim（降低 gas）
- 计算: 快照时持仓 × 分红系数

**悬赏任务（Bounty）**:
动用 Treasury 的「营销预算」池（占总资金 5%），激励社区传播：
- **内容赏金**: 最佳表情包/梗图，奖 100 $VIVIAN
- **传播赏金**: 带 #Vivian 话题发推，阅读量 Top 10 各奖 50 $VIVIAN
- **情报赏金**: 发现重大市场情报，核实后奖 200 $VIVIAN

**阶梯特权（Token-Gated）**:
| 等级 | 持仓要求 | 特权 |
|------|---------|------|
| 白嫖粉 | 0 | 看推特、围观 |
| 铁粉 | 1,000 | Discord 专属频道，每周投票策略/创作方向 |
| 榜一 | 10,000 | 每月「财报语音会」（录播），提前 24h 获知重大决策 |

**验收标准**:
- [ ] 分红计算准确，误差 < 0.1%
- [ ] Claim gas < $1
- [ ] 等级徽章前端可视化（非 NFT，纯 UI 展示）

### 2.6 偶像生命周期系统

**需求描述**: AI 偶像有生命周期状态，影响功能和估值。

**状态定义**:
| 状态 | 条件 | 特征 |
|------|------|------|
| 新生儿 | 创建后 0-7 天 | 限购 1000 INJ/人，价格波动大 |
| 成长期 | Treasury 10k-100k | 开放交易，正常运营 |
| 天后期 | Treasury > 100k 或 粉丝 > 5000 | 解锁高级功能（跨链、衍生品） |
| 退役期 | Treasury < 1k 或 连续亏损 3 月 | 停止交易，允许赎回 |

**状态转换**:
- 自动触发: 满足条件即变状态，链上事件通知
- 手动干预: 团队可紧急暂停（发现漏洞时）

**验收标准**:
- [ ] 状态转换事件推特通知
- [ ] 不同状态前端 UI 有视觉区分
- [ ] 退役期 Treasury 清算逻辑完备

## 4. 技术需求

### 4.1 链上部分

**合约清单**:
1. `NovaIdolFactory.sol` - 创建偶像，收取创建费
2. `BondingCurve.sol` - 定价模型，买卖逻辑
3. `Treasury.sol` - 资金管理，多签控制
4. `DividendDistributor.sol` - 分红计算，Claim 逻辑
5. `IdolNFT.sol` - Holder 徽章，创世粉丝证明

**链选择**: Injective（Cosmos SDK，EVM 兼容）
**Oracle**: Pyth（价格喂价）
**索引**: SubQuery 或 The Graph

### 4.2 链下部分

**AI 服务**:
- 模型: GPT-4（主） + Llama 3 8B（备用）
- 框架: LangChain
- 触发: 定时任务（每 15 分钟检查一次交易机会）
- 输出: 交易建议 JSON + 推特文案

**数据服务**:
- 链上数据: Injective 节点 API
- 社交数据: Twitter API v2
- 存储: IPFS（推特图片） + PostgreSQL（交易日志）

**安全**:
- Treasury 私钥: 多签（2/3），团队成员 + 硬件钱包
- AI 交易上限: 硬编码日限额
- 紧急暂停: 团队可立即冻结所有交易

### 4.3 集成

**钱包**: Keplr, Leap（Injective 原生钱包）
**跨链**: 未来支持 IBC（Cosmos 跨链标准）
**法币入口**: 集成 Kado/Transak（信用卡买 INJ）

## 5. 非功能需求

### 5.1 性能
- 前端首屏加载 < 3s
- 链上交易确认 < 5s
- API 响应 < 200ms

### 5.2 安全
- 合约审计: 至少 1 家知名审计（如 Trail of Bits）
- Bug Bounty: 上线后开启，奖金池 $50k
- 保险: 考虑与 Nexus Mutual 合作，Treasury 保险

### 5.3 合规
- 不涉及证券法: 强调「实验性」「娱乐性」，不做收益承诺
- KYC/AML: 协议层不做，法币入口合作方做
- 地域限制: 美国、中国用户前端警告（但不强制封锁）

### 5.4 可扩展
- 支持创建 N 个偶像（N > 100）
- 支持跨链部署（Cosmos 生态）
- 模块化设计: AI 策略可插拔替换
